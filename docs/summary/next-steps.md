# Next Steps

After completing this sizing guide, follow these steps to proceed with your F5 Distributed Cloud deployment.

---

## 1. Review and Validate

### Internal Review

- [ ] Review all completed sections for accuracy
- [ ] Validate traffic and bandwidth estimates with your infrastructure team
- [ ] Confirm application inventory is complete
- [ ] Verify compliance requirements with your security/legal teams

### Technical Validation

- [ ] Confirm network topology diagrams are accurate
- [ ] Validate IP addressing and subnet information
- [ ] Verify cloud account access and permissions
- [ ] Confirm infrastructure resource availability for CE sites

---

## 2. Export and Submit

### Generate PDF

1. Use your browser's print function (Ctrl+P / Cmd+P)
2. Select **Save as PDF** as the destination
3. Name the file: `[CompanyName]-F5XC-Sizing-[Date].pdf`

Alternatively, download the pre-generated PDF from the documentation site if available.

### Submit to F5

Send the completed PDF to:

- Your **F5 Account Manager**
- Your **F5 Solutions Engineer**

Or upload via your F5 customer portal if available.

---

## 3. Proposal Process

After submission, your F5 team will:

| Step | Timeline | Description |
|------|----------|-------------|
| 1. Review | 2-3 days | F5 reviews sizing requirements |
| 2. Questions | 1-2 days | Follow-up clarification if needed |
| 3. Proposal | 3-5 days | Formal proposal with pricing |
| 4. Review | As needed | Joint review and adjustments |
| 5. Finalize | As needed | Final proposal and agreement |

---

## 4. Pre-Deployment Planning

While waiting for your proposal, prepare for deployment:

### Technical Preparation

- [ ] **Network readiness** - Ensure firewall rules allow F5 XC connectivity
- [ ] **DNS planning** - Plan domain delegation or record updates
- [ ] **Certificate preparation** - Gather custom certificates if using
- [ ] **Cloud credentials** - Set up cloud provider access for F5

### Infrastructure Preparation

- [ ] **CE site resources** - Provision VMs or hardware for Customer Edge
- [ ] **Cloud accounts** - Enable F5 XC access to cloud subscriptions
- [ ] **Network connectivity** - Verify internet access from CE sites

### Team Preparation

- [ ] **Identify stakeholders** - Technical, security, and business contacts
- [ ] **Schedule training** - Plan F5 XC console training
- [ ] **Change management** - Prepare change control processes

---

## 5. Deployment Phases

A typical F5 XC deployment follows these phases:

### Phase 1: Foundation (Weeks 1-2)

- Tenant provisioning
- User access setup
- Initial site deployments (CE or Cloud)
- Basic connectivity validation

### Phase 2: Security Services (Weeks 2-4)

- Load balancer configuration
- WAF policy deployment
- API security setup
- Bot defense activation

### Phase 3: Advanced Features (Weeks 4-8)

- Multi-cloud networking
- Advanced security policies
- Observability integration
- Performance optimization

### Phase 4: Production (Ongoing)

- Traffic migration
- Monitoring and tuning
- Operational handoff
- Continuous optimization
